## üéØ Definition of Done

### For Features

A feature is "done" when:

1. **Functionality**
   - [ ] All acceptance criteria met
   - [ ] Edge cases handled
   - [ ] Error states designed and implemented
   - [ ] Feature flags configured (if needed)

2. **Quality**
   - [ ] Unit tests > 80% coverage
   - [ ] Integration tests passing
   - [ ] E2E tests for critical paths
   - [ ] No critical or high bugs

3. **Performance**
   - [ ] Meets performance budget
   - [ ] Mobile-optimized
   - [ ] Accessibility checked

4. **Documentation**
   - [ ] Code documented
   - [ ] User help articles (if needed)
   - [ ] API documentation updated
   - [ ] CHANGELOG updated

5. **Deployment**
   - [ ] Deployed to staging
   - [ ] QA tested
   - [ ] Deployed to production
   - [ ] Monitoring confirmed

### For Engagements

An engagement is "done" when:

1. **Deliverables**
   - [ ] All deliverables approved by client
   - [ ] Final versions stored in portal
   - [ ] PDFs generated and shared

2. **Knowledge Transfer**
   - [ ] Documentation delivered
   - [ ] Training completed (if applicable)
   - [ ] Handoff meeting conducted

3. **Administrative**
   - [ ] Final invoice sent
   - [ ] Payment received
   - [ ] Engagement closed in system
   - [ ] Retrospective completed

4. **Relationship**
   - [ ] Satisfaction survey sent
   - [ ] Testimonial requested
   - [ ] Referral request made
   - [ ] Future opportunities discussed
